BGRemover API

Eliminación de fondo con IA — REST API v1

Introducción

La API recibe una imagen, elimina el fondo usando modelos de segmentación semántica (rembg) y devuelve un PNG con transparencia.

Base URL: https://bgremover.tutansoft.com/api/v1

Autenticación

Todos los endpoints excepto /health requieren un token de API. Enviarlo en uno de estos headers:

Authorization: Bearer TU_TOKEN_AQUI
# o bien
X-API-Key: TU_TOKEN_AQUI
Sin token configurado en el servidor la API es pública. En producción siempre definir BGREMOVER_API_TOKEN en el servicio systemd.

Endpoints

GET /api/v1/health

Liveness probe. No requiere autenticación. Útil para monitoreo externo.

Respuesta 200

{
  "status":  "ok",
  "service": "bgremover"
}
GET /api/v1/models

Lista los modelos de IA disponibles. Requiere autenticación.

Respuesta 200

{
  "default": "isnet-general-use",
  "models": [
    { "id": "isnet-general-use", "description": "ISNet — best general-purpose quality (default)" },
    { "id": "u2net",            "description": "U²Net — fast, good quality" },
    { "id": "u2netp",           "description": "U²Net-p — lightweight, lower accuracy" },
    { "id": "silueta",          "description": "Silueta — optimised for portraits / people" }
  ]
}
POST /api/v1/remove-bg

Procesa una imagen y devuelve el PNG sin fondo como descarga. Requiere autenticación.

Content-Type del request: multipart/form-data

Parámetros

CampoTipoRequeridoDescripción
image file SÍ Imagen a procesar. Formatos aceptados: JPG, PNG, WEBP, BMP. Tamaño máximo: 20 MB.
threshold int no Umbral de alpha para limpiar artefactos semitransparentes. Rango 0–255. Default: 20. Valores más altos = bordes más agresivos.

Respuesta exitosa (200)

Devuelve el archivo PNG directamente como application/octet-stream con header Content-Disposition: attachment.

El nombre del archivo descargado es el original con sufijo _no_bg.png. Ej: foto.jpg → foto_no_bg.png.

Códigos de error

CódigoCausa
400Imagen ausente, formato no soportado o archivo corrupto.
401Token ausente o inválido.
413Imagen supera el límite de 20 MB configurado en nginx.
422La imagen es válida pero el modelo no pudo procesarla.
500Error interno del servidor.

Todos los errores devuelven JSON:

{ "error": "descripción del error" }

Ejemplos

curl

# Quitar fondo con threshold por defecto
curl -X POST https://bgremover.tutansoft.com/api/v1/remove-bg \
     -H "Authorization: Bearer TU_TOKEN" \
     -F "image=@foto.jpg" \
     --output sin_fondo.png

# Con threshold personalizado
curl -X POST https://bgremover.tutansoft.com/api/v1/remove-bg \
     -H "X-API-Key: TU_TOKEN" \
     -F "image=@foto.jpg" \
     -F "threshold=40" \
     --output sin_fondo.png

PHP

$ch = curl_init('https://bgremover.tutansoft.com/api/v1/remove-bg');
curl_setopt_array($ch, [
    CURLOPT_POST           => true,
    CURLOPT_HTTPHEADER     => ['Authorization: Bearer TU_TOKEN'],
    CURLOPT_POSTFIELDS     => [
        'image'     => new CURLFile('/ruta/foto.jpg'),
        'threshold' => 20,
    ],
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_TIMEOUT        => 120,
]);
$png = curl_exec($ch);

Python

import requests

with open("foto.jpg", "rb") as f:
    response = requests.post(
        "https://bgremover.tutansoft.com/api/v1/remove-bg",
        headers={"Authorization": "Bearer TU_TOKEN"},
        files={"image": f},
        data={"threshold": 20},
        timeout=120,
    )

with open("sin_fondo.png", "wb") as out:
    out.write(response.content)

JavaScript (fetch)

const form = new FormData();
form.append('image', fileInput.files[0]);
form.append('threshold', '20');

const res = await fetch('https://bgremover.tutansoft.com/api/v1/remove-bg', {
  method: 'POST',
  headers: { 'Authorization': 'Bearer TU_TOKEN' },
  body: form,
});
const blob = await res.blob();
const url  = URL.createObjectURL(blob); // usar como src de <img>
